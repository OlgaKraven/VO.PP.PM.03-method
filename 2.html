2.1. Понятие клиентской веб‑разработки

Клиентская веб‑разработка (frontend) — это область веб‑технологий, отвечающая за создание визуального и интерактивного интерфейса, с которым взаимодействует пользователь. Она реализуется в браузере с помощью HTML (структура), CSS (стили) и JavaScript (поведение). Основная задача фронтенда — преобразовать данные, полученные от сервера, в удобный и доступный для восприятия интерфейс.

2.2. HTML5 и семантическая структура документа

Определение

HTML (HyperText Markup Language) — язык гипертекстовой разметки, описывающий структуру и содержание веб‑страницы. Версия HTML5 ввела новые семантические элементы, повышающие читаемость кода и доступность для поисковых систем и технологий ассистивного доступа.

Базовая структура HTML‑документа

<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Театр — репертуар и билеты</title>
  </head>
  <body>
    <header>
      <h1>Театр «АртДом»</h1>
      <nav>
        <ul>
          <li><a href="#repertoire">Репертуар</a></li>
          <li><a href="#tickets">Билеты</a></li>
          <li><a href="#contacts">Контакты</a></li>
        </ul>
      </nav>
    </header>

    <main id="repertoire">
      <section>
        <h2>Текущие спектакли</h2>
        <article>
          <h3>Ромео и Джульетта</h3>
          <p>Постановка в классическом стиле. В главных ролях — ...</p>
          <button>Купить билет</button>
        </article>
      </section>
    </main>

    <footer>
      <p>© 2025 Театр «АртДом»</p>
    </footer>
  </body>
</html>

Комментарий. Этот пример демонстрирует семантическую структуру:

  • <header> — верхняя часть страницы, логотип и навигация.
  • <nav> — блок ссылок.
  • <main> — основное содержимое страницы.
  • <section> — логический раздел (например, афиша).
  • <article> — самостоятельная публикация (описание спектакля).
  • <footer> — нижний блок с контактами и копирайтом.

2.4. Иерархия элементов

HTML‑документ имеет дерево DOM (Document Object Model), где каждый элемент — это узел, вложенный в другие.

graph TD
  A[html]
  A --> B[head]
  A --> C[body]
  C --> D[header]
  C --> E[main]
  C --> F[footer]
  E --> G[section]
  G --> H[article]
  H --> I[h3]
  H --> J[p]

Интерпретация. DOM‑дерево позволяет JavaScript обращаться к любому элементу, изменять его содержимое, атрибуты и стиль.

2.5. Основные элементы HTML и их назначение

1) Заголовки и текстовые блоки

<h1>Главный заголовок</h1>
<h2>Подзаголовок</h2>
<p>Абзац основного текста.</p>
  • <h1>–<h6> — иерархия заголовков (от главного к подзаголовкам).
  • <p> — параграф текста.

Совет: Используйте только один <h1> на страницу — это улучшает SEO и доступность.

2) Списки

<ul>
  <li>Пункт 1</li>
  <li>Пункт 2</li>
</ul>

<ol>
  <li>Первый</li>
  <li>Второй</li>
</ol>
  • <ul> — маркированный список.
  • <ol> — нумерованный список.
  • <li> — элемент списка.

3) Таблицы

<table>
  <caption>Расписание спектаклей</caption>
  <thead>
    <tr><th>Дата</th><th>Спектакль</th><th>Время</th></tr>

Правило: Используйте таблицы только для структурированных данных, а не для вёрстки.

4) Формы

<form action="/submit" method="POST">
  <label for="name">Ваше имя:</label>
  <input id="name" name="name" required>
  <label for="email">Email:</label>
  <input id="email" type="email" name="email" required>
  <button type="submit">Отправить</button>

  • Атрибуты action и method определяют, куда и как отправляется форма.
  • Атрибут required делает поле обязательным.
  • Используйте <label> для доступности (экранные дикторы читают текст метки).

5) Мультимедиа‑вставки

<figure>
  <img src="poster.jpg" alt="Афиша спектакля 'Ромео и Джульетта'">
  <figcaption>Премьера — 4 ноября</figcaption>

<audio controls>

Совет: Добавляйте атрибут alt к изображениям — это важно для SEO и доступности (WCAG).

2.6. Семантика и доступность (Accessibility)

Семантическая разметка делает страницу понятной не только людям, но и машинам:

  • поисковым роботам (SEO);
  • экранным дикторам (a11y);
  • парсерам и индексаторам контента.

Принципы доступной разметки

  1. Каждому разделу — свой смысловой тег (<header>, <main>, <footer>).
  2. Изображения — с alt и <figure>.
  3. Формы — с <label> и aria-* атрибутами.
  4. Навигация — через <nav> и логичные ссылки.
  5. Таблицы — с <caption>, <thead>, <th> для заголовков колонок.

2.7. CSS3 и адаптивная вёрстка

2.7.1. Понятие и назначение CSS3

CSS (Cascading Style Sheets) — каскадные таблицы стилей, язык описания внешнего вида HTML‑документа. Версия CSS3 расширила возможности дизайна: появились Flexbox, Grid Layout, анимации, медиазапросы и переменные.

Итог: цель CSS — отделить структуру (HTML) от представления (внешний вид). Это повышает гибкость, переиспользуемость и доступность веб‑интерфейса.

2.7.2. Базовый синтаксис CSS

селектор {
  свойство: значение;
}
/* Пример */
body {
  font-family: Arial, sans-serif;
  background-color: #fafafa;
  color: #222;
}
  • селектор — элемент HTML, к которому применяются стили;
  • свойство — аспект внешнего вида (цвет, размер, отступ);
  • значение — конкретное значение свойства.

2.7.3. Способы подключения CSS

  1. Внешний файл — лучший способ (удобно поддерживать):
    <link rel="stylesheet" href="style.css">
  2. Встроенные стили — внутри тега <style>:
    <style>p { color: blue; }</style>
  3. Инлайн‑стили — прямо в элементе (не рекомендуется):
    <p style="color: blue;">Текст</p>

2.7.4. Позиционирование элементов

A) Flexbox (Flexible Box Layout)

<div class="menu">
  <div>Главная</div>
  <div>Репертуар</div>
  <div>Контакты</div>
</div>
.menu {
  display: flex;
  justify-content: space-around; /* распределение по оси X */
  align-items: center;           /* выравнивание по оси Y */
  background: #f8f8f8;
}

Ключевые свойства Flexbox:

  • display: flex — включает flex‑режим
  • flex-direction — направление оси (row/column)
  • justify-content — распределение по главной оси
  • align-items — выравнивание по поперечной оси
  • flex-wrap — перенос строк
  • gap — расстояние между элементами

B) CSS Grid Layout

<div class="grid">
  <header>Шапка</header>
  <nav>Меню</nav>
  <main>Контент</main>
  <aside>Боковая панель</aside>
  <footer>Футер</footer>
</div>
.grid {
  display: grid;
  grid-template-areas:
    "header header"
    "nav main"
    "aside main"
    "footer footer";
  grid-template-columns: 1fr 3fr;
  grid-template-rows: auto 1fr auto;
  gap: 10px;
}

header { grid-area: header; background: #ececec; }
nav    { grid-area: nav;    background: #fafafa; }
main   { grid-area: main;   background: #fff; }
aside  { grid-area: aside;  background: #f2f2f2; }
footer { grid-area: footer; background: #ddd; }

Grid подходит для:

  • лендингов, панелей, каталогов;
  • адаптивных макетов с фиксированными и гибкими областями.

2.8. JavaScript и работа с DOM

2.8.1. Назначение JavaScript

JavaScript (JS) — язык программирования, выполняемый в браузере, позволяющий делать веб‑страницы интерактивными: реагировать на действия пользователя, изменять содержимое без перезагрузки, обращаться к серверу, работать с анимациями, формами и API. Современный стандарт — ECMAScript 2015 (ES6) и выше — добавил новые возможности: стрелочные функции, let/const, классы, модули, шаблонные строки и деструктуризацию.

2.8.2. Базовый синтаксис ES6+

// Объявление переменных
let count = 0;       // можно изменять
const pi = 3.14;     // константа, нельзя переопределить
var oldStyle = true; // устаревший способ (лучше избегать)

// Типы данных: number, string, boolean, object, array, function, null, undefined

// Пример операций
let name = "Анна";
let tickets = 3;
console.log(`${name} купила ${tickets} билета(ов)`); // шаблонная строка ES6

2.8.3. Условные операторы и циклы

// Условие
if (tickets > 0) {
  console.log("Есть билеты!");
} else {
  console.log("Распродано");
}

// Циклы
for (let i = 1; i <= 5; i++) {
  console.log(`Билет №${i}`);
}

// Массив + forEach
const shows = ["Гамлет", "Щелкунчик", "Ромео"];
shows.forEach(title => console.log(`Спектакль: ${title}`));

// Совет: используйте стрелочные функции (() => {}) — они компактны и не теряют контекст this.

2.8.4. Функции и стрелочный синтаксис

// Обычная функция
function greet(name) {
  return `Добро пожаловать, ${name}!`;
}

// Стрелочная функция
const greetUser = name => `Здравствуйте, ${name}!`;

console.log(greetUser("Анна"));
  • стрелочные функции короче;
  • не создают собственный this, что удобно для обработчиков событий и колбэков.

2.8.5. Работа с DOM (Document Object Model)

DOM — объектная модель HTML‑документа, представляющая его как дерево узлов. JavaScript позволяет изменять DOM: добавлять, удалять и изменять элементы, атрибуты, стили и текст.

Основные методы доступа к элементам

getElementsByClassName()document.getElementsByClassName("card")По классу (HTMLCollection)

Изменение содержимого и стилей

<h2 id="title">Репертуар</h2>
const title = document.getElementById("title");
title.textContent = "Актуальные спектакли";  // изменение текста
title.style.color = "darkred";               // изменение стиля

Добавление и удаление элементов

const list = document.querySelector("#shows");
const newItem = document.createElement("li");
newItem.textContent = "Щелкунчик";
list.appendChild(newItem);  // добавляем в список
list.removeChild(list.firstElementChild); // удаляем первый элемент

2.8.6. Работа с событиями

<button id="buyBtn">Купить билет</button>
<p id="status"></p>
const button = document.getElementById("buyBtn");
const status = document.getElementById("status");

button.addEventListener("click", () => {
  status.textContent = "Билет добавлен в корзину!";
  status.style.color = "green";

Комментарий: addEventListener() — универсальный способ отслеживать действия пользователя (click, input, change, submit, mouseover, keydown и др.).

2.8.7. Манипуляции с формами

<form id="orderForm">
  <input type="text" id="name" placeholder="Ваше имя" required>
  <button type="submit">Оформить заказ</button>
</form>
<p id="message"></p>
document.getElementById("orderForm").addEventListener("submit", event => {
  event.preventDefault(); // предотвращаем перезагрузку
  const userName = document.getElementById("name").value;
  document.getElementById("message").textContent = `Спасибо, ${userName}! Заказ принят.`;
});

Важно: event.preventDefault() отключает стандартное поведение формы (перезагрузку страницы).

2.8.8. Пример: динамическая афиша театра

<ul id="repertoire"></ul>
<button id="add">Добавить спектакль</button>
const shows = ["Гамлет", "Отелло", "Щелкунчик"];
const list = document.getElementById("repertoire");
const add = document.getElementById("add");

// функция для отображения списка
const render = () => {
  list.innerHTML = "";
  shows.forEach(title => {
    const li = document.createElement("li");
    li.textContent = title;
    list.appendChild(li);
  });
};

// обработчик добавления

  if (newShow) {
    shows.push(newShow);
    render();
  }
});

render();

Результат: При нажатии кнопки появляется окно ввода, список спектаклей обновляется без перезагрузки страницы — демонстрация манипуляции DOM.

2.8.10. Асинхронное программирование

2.8.10.1. Понятие асинхронности

Асинхронное программирование — это подход, позволяющий выполнять длительные операции (запросы к серверу, загрузку файлов, чтение данных) без блокировки основного потока. В JavaScript есть только один поток исполнения — но благодаря событийному циклу (event loop) и колбэкам, JS может обрабатывать несколько задач одновременно.

2.8.10.2. Основные понятия

ТерминНазначение
PromiseОбъект, представляющий результат асинхронной операции (fulfilled / rejected / pending)
fetch()Встроенный метод для отправки HTTP‑запросов (GET, POST и т.д.)
async/awaitСинтаксический сахар для удобной работы с Promise
JSONФормат обмена данными между клиентом и сервером

2.8.10.3. Promise — базовая единица асинхронности

const promise = new Promise((resolve, reject) => {
  const success = true;
  if (success) resolve("Данные успешно получены");
  else reject("Ошибка загрузки");
});

promise
  .then(result => console.log(result))
  .catch(error => console.error(error))
  .finally(() => console.log("Запрос завершён"));

2.8.10.4. Метод fetch()

fetch("shows.json")
  .then(response => response.json()) // преобразуем ответ в объект
  .then(data => console.log(data))
  .catch(error => console.error("Ошибка загрузки:", error));
// Пример POST‑запроса
fetch("https://api.example.com/tickets", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ event_id: 5, seat: "A12" })
})
  .then(res => res.json())
  .then(data => console.log("Ответ сервера:", data))
  .catch(err => console.error("Ошибка:", err));

2.8.10.5. async/await — современный синтаксис

2.9. Модульность и организация кода

2.9.1. Понятие модульности

Модульность — принцип проектирования программ, при котором система делится на независимые части (модули), каждая из которых выполняет конкретную функцию и может разрабатываться, тестироваться и обновляться отдельно.

  • повышает читаемость и повторное использование кода;
  • упрощает сопровождение;
  • изолирует ошибки;
  • обеспечивает масштабируемость проекта.

2.9.2. Основные принципы организации кода

ПринципСутьПример
SRP (Single Responsibility)Каждая функция или модуль выполняет одну задачуrenderTickets() отвечает только за отрисовку
DRY (Don’t Repeat Yourself)Не дублировать код, использовать функции и общие модуливынести повторяющийся код в utils.js
KISS (Keep It Simple, Stupid)Избегать излишней сложностипростая логика вместо «умных» конструкций
Separation of ConcernsРазделение ответственности: данные, логика, интерфейсdata.js, controller.js, view.js

// 1) data.js
export async function getRepertoire() {
  const res = await fetch("shows.json");
  return await res.json();
}

// 2) view.js
export function renderShows(shows) {
  const list = document.getElementById("repertoire");
  list.innerHTML = "";
  shows.forEach(show => {
    const li = document.createElement("li");
    li.textContent = `${show.title} — ${show.date}`;
    list.appendChild(li);
  });
}

// 3) controller.js
import { getRepertoire } from "./data.js";
import { renderShows } from "./view.js";
export async function init() {
  const shows = await getRepertoire();
  renderShows(shows);
}

// 4) app.js
import { init } from "./controller.js";
document.addEventListener("DOMContentLoaded", init);
